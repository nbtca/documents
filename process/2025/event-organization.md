## 写在前面

### 为什么办活动

- 为同学们提供第二课堂学分；
- 为同学创造更多的获得志愿者时长的机会；
- 扩大社团影响力；
- 提升社员的组织能力和技术水平；
- 服务全校同学，践行计算机协会的宗旨。

### 计协办过的活动

- 网络安全之电信诈骗讲座；
- 电脑使用和维护讲座（CA101）；
- 电脑维修日（义务维修）。

:::info 以往活动的经验

- **CA101 讲座**：面向全校新生，内容涵盖基础电脑使用技巧和常见问题处理。讲座形式对场地要求较高，需要多媒体教室和投影设备。建议安排 1-2 名主讲人，控制时长在 1.5 小时以内。
- **维修日**：是计协最具特色的品牌活动。需要提前准备工具箱、备用配件（硅脂、螺丝刀套装等），并安排足够的维修人员。通常选择周末举办，每次持续 3-4 小时。
- **网络安全讲座**：可邀请校内外专家主讲，提前协调好演示设备和网络环境。

:::

## 活动策划

### 确定活动方案

在确定举办活动之前，需要先明确以下几点：

| 事项     | 说明                                                       |
| -------- | ---------------------------------------------------------- |
| 活动目的 | 明确活动要达成的目标                                       |
| 活动形式 | 讲座、维修日、工作坊、比赛等                               |
| 目标受众 | 面向社员、面向全校、面向特定学院等                         |
| 时间安排 | 选择不与考试周、校级活动冲突的时间                         |
| 场地需求 | 教室大小、设备要求（参见[借教室流程](./borrow-classroom)） |
| 人员安排 | 主讲人、现场工作人员、签到人员等                           |
| 预算估计 | 物料费、宣传费等（如有）                                   |

### 撰写策划书

策划书是活动审批的核心材料，需要提交给校团委或数据学院团委审批。策划书应包含以下内容：

1. **活动名称**
2. **主办单位**：宁波财经学院计算机协会
3. **活动背景和目的**
4. **活动时间和地点**
5. **活动流程**：按时间线详细列出每个环节
6. **参与人员**：组织者和预计参与者
7. **物资清单**
8. **预算明细**（如有）
9. **应急预案**：设备故障、人员不足等突发情况的处理方案

:::tip 策划书撰写建议

- 可以参考以往活动的策划书作为模板，在社团存档中查找。
- 策划书中的时间安排要留有余量，避免流程过于紧凑。
- 应急预案不需要太复杂，但至少要考虑到核心环节出问题时的替代方案。

:::

### 提交审批

将策划书提交给对应的审批部门：

- **院级活动**：提交给数据学院团委
- **校级活动**：提交给校团委

:::warning 审批注意事项

- 提交审批后需**主动跟进进度**，不要提交后就不管了。
- 如果策划书被退回修改，及时按意见调整后重新提交。
- 审批通常需要 3-5 个工作日，务必预留充足时间。

:::

## 活动准备阶段

### 场地准备

教室借用流程详见[借教室](./borrow-classroom)。

### 宣传推广

- **线上宣传**：通过社团公众号、QQ 群、朋友圈等渠道发布活动信息
- **线下宣传**：如有需要可制作海报张贴
- 宣传内容应包含：活动名称、时间、地点、面向对象、参与方式

:::tip 宣传时机

- 建议在活动前 **3-5 天**开始宣传，太早容易被遗忘，太晚覆盖面不够。
- 活动前一天可以在群里再发一次提醒。

:::

### 物资准备

根据活动类型准备所需物资：

**讲座类活动：**

- 演示文稿（PPT）
- 翻页笔
- 签到表 / 签到二维码
- 饮用水（如有预算）

**维修日：**

- 螺丝刀套装、撬棒等维修工具
- 备用硅脂、散热贴
- 系统安装 U 盘（PE 系统 + Windows 镜像）
- 签到表 / 登记表
- 维修登记单（记录机主信息和维修内容）

### 人员分工

活动前确认以下角色的负责人：

| 角色         | 职责                               |
| ------------ | ---------------------------------- |
| 活动总负责   | 统筹全局，处理突发情况             |
| 签到人员     | 负责参与者签到、记录出席信息       |
| 主讲人       | 讲座类活动的主讲（如适用）         |
| 现场工作人员 | 维持秩序、引导参与者、协助活动进行 |
| 摄影         | 拍摄活动照片，用于宣传和存档       |

## 活动当天

### 活动前

- [ ] 提前 1 小时确认教室是否开门
- [ ] 检查教室设备（投影仪、麦克风、电源等）
- [ ] 布置场地（桌椅摆放、物资摆放、签到台设置）
- [ ] 工作人员到位并确认各自分工
- [ ] 准备好签到工具

### 活动进行中

- [ ] 安排专人负责签到
- [ ] 按策划书流程推进活动
- [ ] 拍摄活动照片（至少包含全景、特写、互动场景）
- [ ] 注意控制时间，避免严重超时
- [ ] 处理突发情况时保持冷静

### 活动结束后

- [ ] 恢复教室原状（桌椅归位、清理垃圾）
- [ ] 关闭教室设备、电源和门窗
- [ ] 收集整理签到记录
- [ ] 清点归还物资

## 活动后续工作

### 第二课堂学分申报

现在基本没有纸质小票，通过签到名单导入。

- 院级**晚自修可统一请假**。
- 校级大一晚自修需要自行请假。

由于计算机协会是**院级社团**，对应申报表及名单直接找相应部门转交即可：

- 院级学分直接找**社团部**；
- 校级学分找**素拓部**。

申报表模板可能有改动，以素拓部的最新版本为准。

详细的学分申报流程请参见[申请第二课堂学分](./apply-for-credits)。

### 活动招募志愿者相关

填好志愿者招募表后找青协秘书处即可。

:::tip 志愿者招募

- 志愿者招募表需要提前填写并提交，不要等到活动前一天才去找青协。
- 招募表中注意写清楚志愿服务的具体内容和时长。

:::

### 费用报销

开发票。报销时需要准备：

- 正规发票（电子发票或纸质发票均可）
- 费用明细清单
- 活动相关的审批材料

:::warning 报销注意

- 所有需要报销的开支**必须事先获得审批**，不要先花钱再补审批。
- 发票抬头和金额要与实际支出一致。
- 及时报销，不要拖到学期末。

:::

### 活动总结与存档

活动结束后建议完成以下工作：

1. **撰写活动总结**：简要回顾活动情况，记录经验和不足
2. **整理活动照片**：筛选优质照片用于宣传和存档
3. **材料归档**：将策划书、签到记录、学分申报材料、照片等统一存档
4. **复盘讨论**：与团队成员简要讨论本次活动的优缺点，为下次活动积累经验

## 活动策划时间线模板

以下是一个典型活动从策划到完成的时间线，供参考：

| 时间节点      | 事项                                   |
| ------------- | -------------------------------------- |
| 活动前 3-4 周 | 确定活动方案，开始撰写策划书           |
| 活动前 2-3 周 | 提交策划书审批，启动教室借用流程       |
| 活动前 1-2 周 | 确认场地，开始宣传推广                 |
| 活动前 1 周   | 确认人员分工，准备物资                 |
| 活动前 3-5 天 | 发布宣传，招募志愿者                   |
| 活动前 1 天   | 最终确认所有准备工作，发送活动提醒     |
| 活动当天      | 执行活动                               |
| 活动后 1-3 天 | 整理签到记录，撰写总结                 |
| 活动后 1-2 周 | 完成学分申报、志愿者时长录入、费用报销 |
