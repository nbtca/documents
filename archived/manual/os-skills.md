## 选择操作系统

选择自己青睐的操作系统，基本上可供选择的操作系统五花八门，个人的使用习惯和对操作系统的要求最终会影响操作系统的选择。

例如：由于不同开发人员的倾向，Windows 成为了最受游戏欢迎的平台，Linux 成为了服务器管理人员的首选，不同发行版又会受到不同的需求分支多样，至于 macOS 大多是购买了 MacBook 的用户首选；

## 安装操作系统

安装操作系统进入计算机硬件，值得注意的是，对应不同架构的平台，对应操作系统的构建镜像以及构建方法都会略有差异，得益于开发人员的努力，现今绝大多数操作系统的安装变得简单易行，只需要烧录安装介质，按照对应操作系统开发团队的安装手册即可方便的安装；

关于 Windows 系统的安装，请参阅本站 [从零构建好用的Windows](./windows-from-scratch) 文档。

## 使用软件

学习并掌握操作系统及对应平台上的软件解决需求。

计算机最终为需求服务，对应的需要实现这些需求才有的前述对应硬件的搭建、操作系统的搭建等，但是实际实现更加具体的需求又会有具体的专业软件程序作为工具来解决，所以在不同平台安装和使用软件、管理数据也就成为必要学习的部分。

## Windows 文件管理基础

文件管理是操作系统最基本的功能之一，良好的文件管理习惯能够提高效率并减少数据丢失的风险。

### 文件与文件夹

- Windows 使用树状目录结构组织文件，最顶层为各个磁盘分区（C:、D: 等）；
- 每个文件由文件名和扩展名组成（如 `报告.docx`），扩展名决定了文件类型和默认打开方式；
- 建议开启文件扩展名显示：打开文件资源管理器 → 查看 → 勾选「文件扩展名」；
- 常见文件扩展名：`.exe`（可执行程序）、`.docx`（Word 文档）、`.pdf`（PDF 文档）、`.zip`（压缩包）。

:::warning 注意
默认情况下 Windows 隐藏已知文件类型的扩展名，这是恶意软件伪装的常见手段（例如 `照片.jpg.exe` 会显示为 `照片.jpg`）。建议始终开启扩展名显示。
:::

### 文件管理技巧

- **快速搜索文件**：`Win + S` 打开搜索，或在文件资源管理器地址栏直接输入路径；
- **常用快捷键**：`Ctrl+C`（复制）、`Ctrl+X`（剪切）、`Ctrl+V`（粘贴）、`Ctrl+Z`（撤销）、`F2`（重命名）、`Shift+Delete`（永久删除）；
- **文件路径**：右键文件 → 属性，可以查看文件的完整路径，也可以在资源管理器地址栏复制路径；
- **磁盘分区管理**：建议将系统盘（C 盘）和数据盘分开，系统重装时数据盘的内容不受影响。

### 用户文件夹

- Windows 为每个用户创建了专属文件夹，位于 `C:\Users\用户名\` 下；
- 包含桌面、文档、下载、图片、音乐、视频等子文件夹；
- 建议将桌面和下载文件夹迁移到非系统盘：右键文件夹 → 属性 → 位置 → 移动。

## 任务管理器与进程管理

任务管理器是 Windows 中最重要的系统工具之一，掌握它的使用对于故障排查至关重要。

### 打开方式

- `Ctrl + Shift + Esc`：直接打开任务管理器（推荐）；
- `Ctrl + Alt + Delete`：安全屏幕中选择任务管理器；
- 右键任务栏 → 任务管理器。

### 主要功能

#### 进程选项卡

- 显示当前运行的所有程序和后台进程；
- 可以按 CPU、内存、磁盘、网络使用率排序，快速定位资源占用异常的进程；
- 右键进程可以结束任务、打开文件所在位置、搜索在线信息等。

#### 性能选项卡

- 实时显示 CPU、内存、磁盘、网络、GPU 的使用情况和历史曲线；
- CPU 使用率长期 100% 可能是程序异常或恶意软件；
- 内存使用率过高时系统会变得卡顿，考虑关闭不必要的程序或增加物理内存。

#### 启动选项卡

- 管理开机自启动程序，禁用不必要的启动项可以加快开机速度；
- 右键可以禁用/启用启动项，查看启动影响程度。

:::tip 维修提示
电脑运行卡顿时，第一步就是打开任务管理器查看资源占用情况。找到 CPU 或内存占用异常的进程是解决问题的关键。
:::

## 系统设置与控制面板

Windows 同时存在"设置"应用和传统"控制面板"，两者功能有交叉但各有侧重。

### 设置（Settings）

通过 `Win + I` 打开，是 Windows 现代化的系统配置界面，常用功能包括：

- **系统**：显示设置、通知、电源选项、存储管理；
- **网络和 Internet**：WiFi、以太网、VPN、代理设置；
- **个性化**：壁纸、主题、锁屏、任务栏配置；
- **应用**：管理已安装应用、默认应用、启动应用；
- **更新和安全**（Windows 10）/ **Windows 更新**（Windows 11）：系统更新、恢复选项。

### 控制面板（Control Panel）

通过搜索"控制面板"打开，包含一些设置应用中尚未迁移的功能：

- **程序和功能**：卸载程序（比设置中的应用列表更详细）；
- **网络和共享中心**：详细的网络适配器设置和诊断；
- **设备管理器**：查看和管理所有硬件设备及其驱动，是排查硬件问题的重要工具；
- **电源选项**：详细的电源计划配置。

### 设备管理器

设备管理器是维修人员使用频率极高的工具：

- 打开方式：右键开始菜单 → 设备管理器，或 `Win + X` → 设备管理器；
- 设备名称前出现黄色感叹号表示驱动异常；
- 右键设备可以更新驱动、禁用设备、卸载设备；
- 查看隐藏设备可以找到曾经连接过但当前未连接的设备。

## 命令行基础

命令行是通过文本指令操作计算机的方式，在系统管理和故障排查中非常有用。

### CMD（命令提示符）

- 打开方式：搜索"cmd"或 `Win + R` 输入 `cmd`；
- 以管理员身份运行：搜索"cmd"后右键选择「以管理员身份运行」；

常用命令：

```cmd
:: 查看网络配置
ipconfig /all

:: 刷新 DNS 缓存
ipconfig /flushdns

:: 测试网络连通性
ping nbtca.space

:: 查看路由跟踪
tracert nbtca.space

:: 检查系统文件完整性
sfc /scannow

:: 查看磁盘信息
wmic diskdrive list brief

:: 关机/重启
shutdown /s /t 0
shutdown /r /t 0
```

### PowerShell

PowerShell 是 Windows 更强大的命令行工具，兼容大部分 CMD 命令并提供更丰富的功能：

- 打开方式：搜索"PowerShell"或 `Win + X` → Windows 终端；
- PowerShell 使用 cmdlet（如 `Get-Process`、`Get-Service`）格式的命令；
- 推荐使用 Windows Terminal 作为统一的终端界面，支持多标签页和自定义外观。

常用 PowerShell 命令：

```powershell
# 查看所有进程
Get-Process

# 查看系统信息
Get-ComputerInfo

# 查看已安装程序
Get-Package

# 查看磁盘空间
Get-PSDrive -PSProvider FileSystem

# 查看网络适配器
Get-NetAdapter
```

:::tip 提示
Windows 11 默认终端已经改为 Windows Terminal，集成了 PowerShell、CMD 和 WSL 等多个命令行环境，推荐使用。
:::

## 常见故障排查步骤

作为维修队成员，遇到系统问题时可以按照以下思路进行排查。

### 通用排查流程

1. **了解问题现象**：详细询问用户问题的表现、发生频率、是否有规律；
2. **确认最近变更**：询问问题出现前是否安装了新软件、更新了系统或驱动、更改了设置；
3. **检查资源占用**：打开任务管理器查看 CPU、内存、磁盘是否存在异常占用；
4. **检查事件查看器**：`Win + X` → 事件查看器 → Windows 日志，查看系统和应用程序日志中的错误信息；
5. **尝试安全模式**：如果正常模式下无法操作，尝试进入安全模式排查（开机时按 F8 或通过设置进入）；
6. **搜索错误信息**：将具体的错误代码或错误信息在搜索引擎中检索，通常能找到解决方案。

### 常见问题速查

| 问题              | 可能原因                         | 解决方向                                                        |
| ----------------- | -------------------------------- | --------------------------------------------------------------- |
| 开机速度慢        | 启动项过多、硬盘老化             | 禁用不必要的启动项，检查硬盘健康状态                            |
| 系统卡顿          | 内存不足、磁盘占用 100%          | 关闭高占用进程，检查是否有恶意软件                              |
| 蓝屏（BSOD）      | 驱动冲突、内存故障、系统文件损坏 | 记录蓝屏代码，参考微软文档排查                                  |
| 无法连接网络      | 驱动缺失、IP 配置错误、DNS 问题  | 检查网卡驱动，重置网络设置                                      |
| 软件无法安装/运行 | 缺少运行库、权限不足、兼容性问题 | 安装 VC++ 运行库，尝试兼容模式运行                              |
| 磁盘空间不足      | 临时文件堆积、大文件占用         | 参阅本站 [C盘清理标准化流程](../../tutorial/2025/clean-drive-c) |

### 系统修复工具

Windows 内置了多个系统修复工具，维修时常用到：

- **SFC（系统文件检查器）**：在管理员 CMD 中运行 `sfc /scannow`，自动修复损坏的系统文件；
- **DISM（部署映像服务和管理工具）**：运行 `DISM /Online /Cleanup-Image /RestoreHealth`，修复 Windows 组件存储；
- **chkdsk（磁盘检查）**：运行 `chkdsk C: /f /r`，检查并修复磁盘错误；
- **系统还原**：如果之前创建了还原点，可以将系统恢复到正常工作的状态；
- **重置此电脑**：设置 → 系统 → 恢复 → 重置此电脑，可选择保留或删除个人文件。
