## 选择Windows版本

### Windows 历史与各版本概况

下面是 Windows 系列的主要版本演进（略去一些小版本、旧的 DOS 继承版本）以及各版本的主要特点与局限。

| 时代 / 分类                  | 版本名称                 | 发布时间 / 世代 | 内核 / 架构           | 主要特点 / 创新 / 局限                                                                                                               | 是否仍推荐使用 / 支持情况                    |
| ---------------------------- | ------------------------ | --------------- | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------- |
| DOS 继承 / 早期 GUI 阶段     | Windows 1.0              | 1985 年         | 基于 MS-DOS           | 最早的图形界面扩展，窗口不能重叠，只能平铺显示 ([Wikipedia][1])                                                                      | 已完全淘汰，不用                             |
|                              | Windows 2.x / 2.1        | 1987 万         | 基于 MS-DOS           | 增加对重叠窗口、扩展模式支持、更好 UI 控件等 ([Wikipedia][1])                                                                        | 已淘汰                                       |
|                              | Windows 3.0 / 3.1 / 3.11 | 1990 年代初期   | DOS + 保护模式 / 扩展 | 引入真正的多任务、虚拟内存、对 386 的支持、TrueType 字体、改善可用性 ([Wikipedia][1])                                                | 已淘汰                                       |
| “Windows 9x / 家用线”        | Windows 95               | 1995 年         | 混合 / DOS 兼容       | 引入“开始菜单”、任务栏、即插即用、长文件名支持、图形化 UI 标志性转变 ([香港01][2])                                                   | 已淘汰                                       |
|                              | Windows 98 / 98 SE       | 1998 / 1999     | 基于 9x 内核          | 增强硬件支持、性能优化、系统稳定性改善                                                                                               | 已淘汰                                       |
|                              | Windows Me               | 2000 年         | 9x / DOS 混合         | 最后一代基于 DOS 架构的 Windows；兼容性与稳定性较差 ([Wikipedia][3])                                                                 | 已淘汰                                       |
| Windows NT / 企业 / 专业路线 | Windows NT 3.x / 4.0     | 1993–1996       | NT 内核（32 位）      | 更稳健、分离内核 / 用户模式，提高安全性，支持多用户、多任务 ([Wikipedia][1])                                                         | 已淘汰                                       |
|                              | Windows 2000             | 2000 年         | NT 内核               | 面向企业 / 专业用户版本，带有更强的稳定性 / 安全性                                                                                   | 已停止支持                                   |
|                              | Windows XP               | 2001 年         | NT 内核（32 / 64 位） | 大量软件支持、良好的兼容性、长期流行版本                                                                                             | 微软已停止主流支持（延展支持在 2014 年结束） |
|                              | Windows Vista            | 2007 年         | NT 内核               | 新 UI (Aero)、改进安全模型 (UAC)、图形硬件加速，但性能问题 / 驱动兼容问题严重                                                        | 不推荐使用                                   |
|                              | Windows 7                | 2009 年         | NT 内核               | 稳定、兼容性好、资源占用适中，是许多用户认为最成熟的一代 Windows                                                                     | 已停止官方技术支持（扩展支持已结束）         |
|                              | Windows 8 / 8.1          | 2012 / 2013 年  | NT 内核               | 引入 Metro / Modern UI、触控支持、启动性能提升等                                                                                     | 兼容性争议较多，不如 7 / 10 那么稳妥         |
|                              | Windows 10               | 2015 年起       | NT 内核               | 微软开始“Windows 即服务”（持续更新），支持新的硬件（比如 DirectX 12）、混合设备、触控 / 平板 / PC 一体化等 ([Microsoft Learn][4])    | 目前依然是主流选择，微软仍提供更新支持       |
|                              | Windows 11               | 2021 年起       | NT 内核               | 新的 UI 界面（居中开始菜单、圆角窗口、任务视图改进）、对现代硬件要求更高（例如 TPM 2.0、安全启动、较高的 CPU 要求） ([Wikipedia][5]) | 若硬件支持，是未来趋势，微软主推版本         |

此外，微软也有 Windows 的长期支持版本（LTSC / LTSB / Enterprise）线，用于对稳定性要求极高、变动少的环境（如工业控制、银行机房等）。这些版本较少接收新功能更新，但会长期收到安全更新。

- 各版本 / 在 Windows 10 / Windows 11 生态中的不同版本区别（家庭 / 专业 / 企业 / LTSC 等）

在 Windows 10 / 11 中，不同“版本”主要区别在于功能权限和更新策略：

- **家庭版 / Home**：适合普通个人用户，功能较为基础，缺乏一些企业 / 管理相关的高级功能（如组策略、加入域、Hyper-V 虚拟化、远程桌面作为主机等）。
- **专业版 / Pro**：在家庭版基础上增加许多管理、虚拟化和网络功能，适合对电脑有额外需求的用户（比如软件开发、轻度服务器、虚拟机或连接公司网络等）。
- **企业版 / Enterprise**：面向企业用户，功能更全面（例如高级安全、企业管理、分发和更新控制等）。通常由企业授权使用。
- **教育版 / Education**：对教育机构用户开放，有些类似于企业版。
- **LTSC / LTSB（长期服务通道）**：这个版本取消或冻结大多数“功能性更新”，只保留安全更新，界面与功能变化较少，适合稳定性要求极高的场景。

在选购 / 安装时，一般对个人 / 家用用户而言，Home 或 Pro 就足够。

- 各版本的优劣 / 局限总结

优点方面，随着版本推进：

- 对现代硬件支持越来越好（例如最新的 CPU、固态硬盘 NVMe、显卡、USB 规范等）；
- 安全性提升（如内核隔离、驱动签名、安全启动、硬件信任根 TPM、Windows Defender 等）；
- 更新与生态成熟（软件、驱动支持广泛、游戏 / 开发 /办公软件兼容性好）；
- 功能上更现代（虚拟化、容器、子系统、触控 / 平板 / 二合一设备支持更好）；

局限 / 风险方面：

- 新版对硬件要求更高：例如 Windows 11 要求 TPM 2.0、安全启动、较新的 CPU 架构等；

- 升级 / 更新可能带来兼容性问题（某些老设备 / 驱动 /软件可能在新版中出问题）；

- 部分“新功能”可能是用户不需要的，仅增加系统复杂性或资源开销；

- 对于长期稳定性要求高的环境，频繁功能更新可能会干扰（这时 LTSC 版本更适合）。

### 如何在当前环境下选择合适的 Windows 版本

在给出镜像之前，先给你几个选版建议，供你根据自己的实际情况判断。

- 选版建议（考虑以下因素）

1. **硬件兼容性 / 要求**
   - 如果设备较新（CPU 较新、支持 TPM、支持安全启动等），可以考虑 Windows 11。
   - 如果设备较旧，不满足 Windows 11 的硬件门槛，建议选择 Windows 10。
   - 若是台式机 / 老笔记本 /某些 BIOS 不支持 TPM，则 Windows 10 更保险。

2. **用途 / 功能需求**
   - 普通办公、上网、影音娱乐：Home 版或 Pro 版足够。
   - 有远程桌面主机需求、虚拟机 / Hyper-V 需求、加入公司域 / 管理需求：建议 Pro 或 Enterprise。
   - 若希望系统功能稳定、减少新功能更新干扰（如用于控制系统 / 工业 /服务器环境）：考虑 LTSC / LTSB（长期版）。

3. **软件 / 驱动兼容性**
   - 如果你有一些老硬件 /外围设备（打印机、扫描仪、专业设备等），需要确认驱动在目标系统中能否正常工作。
   - 某些专业软件可能在最新系统中尚未完全适配。

4. **更新策略 / 稳定性**
   - Windows 10 能够获得微软支持到某个年限（微软会逐渐结束支持），如果你打算长期使用，可能要考虑未来升级。
   - Windows 11 是未来方向，但早期版本可能还存在一些兼容性问题。
   - 如果你不喜欢频繁“功能更新”的干扰，LTSC 版本是一个折中选择。

5. **个人预算 / 许可**
   - 正版授权成本也可能是一个考虑因素。
   - 如果已有某些授权（家用 / 教育 /企业许可），可以考虑利用已有许可安装相应版本。

- 推荐组合（根据典型场景）

| 场景                                      | 推荐版本                                        | 理由                                     |
| ----------------------------------------- | ----------------------------------------------- | ---------------------------------------- |
| 新笔记本 / 新台式机，硬件较新的情况       | Windows 11 Home / Pro                           | 使用最新系统，界面现代、功能全面         |
| 设备旧一些 / 不满足 Windows 11 要求       | Windows 10 Pro / Home                           | 性能稳定、兼容性好                       |
| 对远程桌面或虚拟化有需求                  | Windows 11 Pro / Windows 10 Pro                 | 支持 Hyper-V、加入域、远程桌面主机等功能 |
| 用于控制系统 / 工业 /对稳定性极高要求环境 | Windows 10 / Windows 11 LTSC / 企业长期支持版本 | 减少功能变动，仅保留安全更新             |
| 希望未来长期使用、减少换系统成本          | 可考虑 Windows 11（硬件满足情况下）             | 微软主推版本，未来生态支持更好           |

总结来说，如果硬件条件允许，我会推荐 **Windows 11 Pro**（对于有高级需求用户）或 **Windows 11 Home / Windows 10 Pro**（对于普通用户）作为首选；如果硬件限制或兼容性风险高，可以先选 Windows 10。

### 官方 / 合法的镜像获取方式与注意事项

安装 Windows 最重要的一点是要获取 **官方纯净原版镜像**，避免使用带有恶意软件 / 破解 / 修改的镜像。下面是几种正规的下载途径和注意事项。

- 官方 / 微软推荐下载方式

1. **微软官方网站下载 Windows 10 ISO**
   微软提供官方的 Windows 10 ISO 下载页面：
   → “下载 Windows 10 光盘映像（ISO 文件）” 页面 ([Microsoft][6])
   你可以使用微软的 Media Creation Tool（媒体创建工具）来下载镜像或制作安装媒体。 ([CSDN Blog][7])

2. **微软官方网站下载 Windows 11 ISO**
   微软官方也提供 Windows 11 的磁盘镜像下载页面：([Microsoft][8])
   在该页面可以选择语言、版本、架构（x64 / ARM）等。 ([Microsoft][8])

3. **微软评估中心 / 企业 / 教育渠道**
   对于 Windows 企业版 /教育版 /长期支持版本（LTSC / LTSB），可以通过微软的评估中心或企业 /教育授权渠道获取合法 ISO 镜像。

4. **微软官方镜像库 / MSDN / Visual Studio 订阅**
   如果你是微软官方开发者 / MSDN 或 Visual Studio 订阅用户，可以直接从微软镜像库获取多个版本的原版 ISO。

- 其他辅助 /镜像索引网站（需谨慎验证）

* 有些社区 /镜像聚合站点（如 “HelloWindows”“系统库” 等）收录微软官方原版镜像资源，供用户下载。需确认其镜像是否未被篡改。
* 有网友推荐的 ISO 聚合站点（如 rg-adguard 等）可以用于查找微软官方镜像下载链接（仅做辅助参考）([Reddit][9])
* 在国内，有些镜像 / 系统资源站点公开收录微软官方原版镜像供下载（如 HelloWindows 等）([hellowindows.cn][10])

⚠ **但必须强调：非微软官方渠道下载时，一定要通过校验（MD5 / SHA-256 / 数字签名）来确保镜像未被篡改或植入恶意内容**。

- 镜像获取地址（示例）

* Windows 10 官方下载（ISO / 媒体创建工具）：微软 “下载 Windows 10 光盘映像” 页面 ([Microsoft][6])
* Windows 11 官方下载（ISO）：微软 “下载 Windows 11” 页面 ([Microsoft][8])
* 某些镜像聚合 / 下载站点（作为备用）：HelloWindows（纯净原版镜像收录站）([hellowindows.cn][10])

此外，国内也有一些技术社区或镜像备案站会同步微软官方镜像，不过在使用时仍要谨慎校验。

### 安装 / 使用中的注意事项

在你选好版本、下载 ISO 之后，下面这些是安装 / 使用时要注意的：

1. **校验镜像**
   使用微软或镜像站提供的校验值（MD5 / SHA256 /数字签名）核对下载的 ISO 文件，确保其完整性与可信性。

2. **制作可启动安装介质**
   通常用 USB U 盘制作启动盘（使用 Microsoft 提供的工具，如 Windows USB/DVD Download Tool，Rufus 等），将 ISO 写入 USB。
   如果 BIOS / UEFI 模式不同（传统 BIOS / UEFI /兼容模式），要注意启动模式设置。

3. **备份数据 / 驱动准备**
   安装前备份当前重要数据。
   如果你的电脑有特殊硬件（网卡、显卡、声卡、指纹、WiFi 模块等），事先准备好这些硬件在目标系统下的驱动程序可能会更安全。

4. **安装顺序 / 分区**
   在安装时注意分区、格式化方案（GPT / MBR）与系统启动模式一致（UEFI 推荐 GPT + EFI 启动）。
   如果有双系统或多硬盘，也要小心不要误格式化其他盘。

5. **激活 / 授权**
   安装后要合法激活系统（使用正版密钥 /数字许可）。不要尝试不合法破解，否则有安全风险且可能违反法律 /微软许可协议。

6. **安装更新 / 驱动**
   第一次上线后，先更新系统补丁、再安装显卡 /芯片组 /外围设备驱动。避免因驱动版本冲突导致的问题。

### 建议总结

- 如果你的电脑硬件条件允许，建议安装 **Windows 11 Pro**（或 Windows 11 Home）作为主系统。
- 如果硬件不符合 Windows 11 要求，或者你担心兼容性风险，使用 **Windows 10 Pro / Home** 是更稳妥的选择。
- 若你更倾向稳定、变动少的系统环境，可考虑 Windows 的 **LTSC / 长期支持版本**。
- 无论安装哪个版本，都务必从微软官方或可信渠道下载原版 ISO，并做完整校验，确保系统安全干净。

## 获取Windows

在上一节中我们介绍了不同版本 Windows 的区别和选择建议，也列出了一些官方 / 可信的镜像来源。这里我们具体说一下如何把 ISO 镜像下载到本地。

### 方法一：微软官网直接下载 ISO

这是最推荐的方式。

1. 打开微软官方下载页面：
   - Windows 10：<https://www.microsoft.com/zh-cn/software-download/windows10>
   - Windows 11：<https://www.microsoft.com/zh-cn/software-download/windows11>
2. 在页面中找到 **"下载 Windows XX 光盘映像（ISO 文件）"** 部分。
3. 选择版本（通常选"Windows XX"即可，会包含家庭版和专业版）。
4. 选择语言为 **简体中文**。
5. 点击 **64 位下载**，开始下载 ISO 文件（通常 4~6 GB）。

:::tip 提示
如果你在 Windows 电脑上访问 Windows 10 下载页面，微软可能会强制你使用 Media Creation Tool 而不显示直接下载链接。你可以按 `F12` 打开开发者工具，切换 User Agent 为 iPad 或 macOS，刷新页面后即可看到直接下载链接。
:::

### 方法二：使用 Media Creation Tool（媒体创建工具）

Media Creation Tool 是微软官方提供的工具，可以下载镜像并直接制作安装 U 盘。

1. 在上述 Windows 10 / 11 下载页面中，点击 **"立即下载工具"**（仅 Windows 10 页面提供）或 **"下载 Windows 11 安装助手"**。
2. 运行下载的 `MediaCreationTool.exe`。
3. 接受许可条款。
4. 选择 **"为另一台电脑创建安装介质"**。
5. 选择语言、版本、架构（通常选 64 位）。
6. 选择目标：
   - **U 盘**：直接写入 U 盘（需要 8GB 以上的空 U 盘）。
   - **ISO 文件**：下载 ISO 到本地，后续自行烧录。

:::warning 注意
使用 Media Creation Tool 写入 U 盘时，U 盘中的所有数据将被清除，请提前备份。
:::

### 方法三：第三方可信镜像站

如果你的网络访问微软官方速度较慢，可以使用以下收录微软原版镜像的站点：

- **HelloWindows**：<https://hellowindows.cn/>，收录微软官方原版镜像，提供 SHA-256 校验值。
- **MSDN, I Tell You（ITELLYOU）**：<https://msdn.itellyou.cn/>（旧站）或 <https://next.itellyou.cn/>（新站），长期收录微软官方原版镜像。

:::warning 安全警告
从任何非微软官方渠道下载的镜像，**务必使用 SHA-256 校验值验证文件完整性**。校验方法如下：

```powershell
# 在 PowerShell 中执行
Get-FileHash "C:\路径\你的镜像.iso" -Algorithm SHA256
```

将输出的哈希值与官方 / 镜像站提供的 SHA-256 值进行比对，完全一致才可使用。
:::

### 方法四：MSDN / Visual Studio 订阅

如果你拥有微软 Visual Studio 订阅（原 MSDN 订阅），可以登录 <https://my.visualstudio.com/Downloads> 下载各种版本的 Windows 镜像，包括企业版和 LTSC 版本。部分高校可能通过校园协议提供这类订阅，可以向学校信息中心咨询。

## 烧录安装介质

拿到 ISO 镜像后，需要将其写入 U 盘，制作可启动的安装介质。以下是几种常用方法。

:::info 准备工作

- 一个容量 **8GB 以上** 的 U 盘（推荐 USB 3.0 以上，速度更快）。
- U 盘中的所有数据将被清除，请提前备份。
- 下载好的 Windows ISO 镜像文件。
  :::

### 方法一：Rufus（推荐）

Rufus 是一款免费、开源、轻量的 USB 启动盘制作工具，支持 UEFI 和传统 BIOS 启动。

1. 下载 Rufus：<https://rufus.ie/zh/>（建议下载便携版，免安装）。
2. 插入 U 盘，运行 Rufus。
3. **设备**：选择你的 U 盘。
4. **引导类型选择**：点击「选择」，找到下载好的 Windows ISO 文件。
5. **分区类型**：
   - 如果你的电脑支持 UEFI 启动（绝大多数 2015 年后的电脑），选择 **GPT**。
   - 如果是较老的电脑只支持传统 BIOS，选择 **MBR**。
6. **目标系统类型**：与分区类型对应，UEFI 选 `UEFI (非 CSM)`，MBR 选 `BIOS (或 UEFI-CSM)`。
7. **文件系统**：保持默认（NTFS）即可。
8. 点击 **开始**，等待写入完成（通常需要 5~15 分钟）。

:::tip 提示
Rufus 在制作 Windows 11 安装盘时会弹出一个对话框，提供跳过 TPM 2.0 / 安全启动检测、跳过微软账户登录等选项。如果你的电脑不满足 Windows 11 硬件要求，可以勾选这些选项。
:::

### 方法二：Ventoy（适合需要多个镜像的用户）

Ventoy 是一款开源工具，安装后只需将 ISO 文件直接复制到 U 盘即可启动，支持同时存放多个镜像。

1. 下载 Ventoy：<https://www.ventoy.net/cn/download.html>。
2. 运行 `Ventoy2Disk.exe`。
3. 选择你的 U 盘，点击 **安装**（首次使用）或 **升级**（已安装过 Ventoy）。
4. 安装完成后，U 盘会被分成两个分区，其中一个大分区用于存放 ISO。
5. 将 Windows ISO 文件直接复制到 U 盘的大分区中。
6. 启动时选择 U 盘，Ventoy 会列出所有 ISO 文件供你选择。

:::tip 提示
Ventoy 的优势在于你可以在同一个 U 盘中同时放入 Windows 10、Windows 11、PE 系统等多个镜像，无需反复烧录。对于维修队成员来说尤其方便。
:::

### 方法三：Media Creation Tool 直接写入

如果你在上一步「获取 Windows」中使用了 Media Creation Tool，可以直接选择写入 U 盘，无需额外工具。具体操作见上一节。

### 验证启动盘

制作完成后，建议确认 U 盘是否可以正常启动：

1. 在文件管理器中打开 U 盘，确认其中有 `setup.exe`（Rufus / MCT 方式）或 ISO 文件（Ventoy 方式）。
2. 对于 UEFI 启动盘（GPT 分区），U 盘中应有 `EFI` 文件夹。
3. 如果有条件，可以在虚拟机中测试 U 盘是否能正常启动。

## 安装Windows

制作好启动 U 盘后，就可以进入安装流程。整个过程分为三个阶段：进入 BIOS 设置启动顺序、执行安装、完成初始设置（OOBE）。

### 第一步：从 U 盘启动

1. 将制作好的启动 U 盘插入目标电脑。
2. 开机或重启，在品牌 Logo 出现时连续按下对应的启动菜单快捷键（各品牌不同）。

:::tip 各品牌启动快捷键
各厂商的 BIOS / Boot Manager 进入方式请参考 [维修操作指南 - 各厂商笔记本 BIOS/Boot Manager 入口](/repair/guide#各厂商笔记本-bios-boot-manager-一键恢复入口)。常见的如：联想 `F12`、华硕 `Esc`、惠普 `F9`、戴尔 `F12`。
:::

3. 在启动菜单中选择你的 U 盘（通常显示为 U 盘品牌名或 `UEFI: USB xxx`）。
4. 如果是 Ventoy，会先进入 Ventoy 菜单，选择对应的 Windows ISO 文件即可。

:::warning 无法识别 U 盘？
如果启动菜单中看不到 U 盘，可能需要进入 BIOS 设置：

- 关闭 **Secure Boot**（安全启动）。
- 将启动模式切换为 **UEFI**（如果 U 盘是 GPT 格式烧录的）。
- 确认 U 盘已正确插入并被识别。
  :::

### 第二步：Windows 安装过程

1. 从 U 盘启动后，会进入 Windows 安装程序界面。
2. **选择语言和区域**：语言选 `中文（简体，中国）`，时间和货币格式选 `中文（简体，中国）`，键盘选 `微软拼音` 或 `美式键盘`。
3. 点击 **"现在安装"**。
4. **输入产品密钥**：如果有密钥可以输入；如果没有，点击 **"我没有产品密钥"**，安装完成后再激活。
5. **选择版本**：如果出现版本选择界面，根据需要选择 Home 或 Pro。
6. 接受许可条款。
7. **安装类型**：选择 **"自定义：仅安装 Windows（高级）"**。

### 第三步：磁盘分区

这是安装过程中最需要注意的一步。

:::warning 数据安全
分区操作会导致数据丢失，请确认已备份重要数据！如果是多硬盘电脑，请仔细辨认目标硬盘，不要误操作。
:::

**全新安装（推荐）：**

1. 在磁盘分区界面，选中目标硬盘上的所有分区，逐个点击 **"删除"**。
2. 删除完成后，选中 **"未分配的空间"**，点击 **"新建"**。
3. 输入系统分区大小（建议至少 **100GB**，即 102400 MB）。
4. Windows 会自动创建 EFI 系统分区、MSR 保留分区等必要分区。
5. 选中刚创建的主分区（通常是最大的那个），点击 **"下一步"**。

**保留其他分区的安装：**

如果硬盘上有 D 盘等数据分区不想丢失，只需删除原来的系统分区（通常是 C 盘和相关的 EFI / 恢复分区），在释放出的未分配空间上新建分区并安装即可。

安装程序会自动复制文件、安装功能和更新，整个过程大约需要 10~30 分钟，期间电脑会自动重启数次。**重启后请拔掉 U 盘**，避免再次从 U 盘启动。

### 第四步：初始设置（OOBE）

安装完成后会进入开箱体验（OOBE）设置界面。

1. **选择地区**：选择 `中国`。
2. **选择键盘布局**：选择 `微软拼音`，第二键盘布局可以跳过。
3. **联网设置**：
   - Windows 11 默认要求联网并使用微软账户登录。
   - 如果你希望使用本地账户，参考下方的跳过方法。
4. **账户设置**：设置用户名和密码。
5. **隐私设置**：建议将所有隐私选项（位置、诊断数据、广告 ID 等）都关闭。
6. 等待系统完成最终配置，进入桌面。

### Windows 11 跳过联网和微软账户

Windows 11 安装时默认强制联网并要求使用微软账户登录。如果你希望创建本地账户，可以使用以下方法：

#### 方法一：命令绕过（推荐）

在 OOBE 联网界面按 `Shift + F10` 打开命令提示符，输入：

```cmd
oobe\bypassnro
```

电脑会自动重启，再次进入 OOBE 后联网界面会出现 **"我没有 Internet 连接"** 的选项，点击即可跳过联网和微软账户。

#### 方法二：Rufus 预设绕过

如果在使用 Rufus 制作启动盘时勾选了"移除 Windows 11 对在线微软帐户的要求"选项，安装时会自动跳过联网要求。

:::tip 建议
对于个人使用，本地账户已经完全足够。使用微软账户登录虽然可以同步设置和 OneDrive，但也会带来额外的隐私数据上传。根据个人偏好选择即可。
:::

## 构建网络环境

全新安装的 Windows 可能无法直接联网，尤其是台式机或部分笔记本缺少网卡驱动的情况。这一步帮助你让电脑先连上网，为后续安装驱动和软件做准备。

### 检查网络状态

安装完成进入桌面后，先检查右下角任务栏的网络图标：

- **Wi-Fi 图标正常**：说明无线网卡驱动已自动安装，直接连接 Wi-Fi 即可。
- **有线连接图标正常**：插入网线即可联网。
- **显示红叉或无网络图标**：说明网卡驱动缺失，需要手动安装。

### 网卡驱动缺失的解决方案

:::warning 常见问题
全新安装 Windows 后没有网卡驱动是很常见的情况，尤其是以下场景：

- 使用 Intel 较新的 Wi-Fi 网卡（如 AX201、AX211、BE200 等）。
- 使用 Realtek 2.5G 有线网卡的台式机。
- 安装的 Windows 版本较老，不包含新硬件的驱动。
  :::

#### 方法一：使用手机 USB 共享网络（最方便）

1. 用数据线将手机连接到电脑。
2. 在手机上开启 **USB 网络共享**：
   - Android：`设置` → `更多连接方式` → `USB 网络共享`（不同品牌路径略有不同）。
   - iPhone：`设置` → `个人热点` → 用数据线连接后自动共享。
3. 电脑会自动识别并联网，无需额外驱动。

#### 方法二：提前准备网卡驱动

在安装系统之前，用另一台能上网的电脑下载好网卡驱动，存放在 U 盘中：

1. 查看目标电脑的网卡型号（可以在设备管理器中查看，或查阅电脑官网的规格参数）。
2. 到网卡厂商官网下载对应驱动：
   - Intel 无线网卡：<https://www.intel.cn/content/www/cn/zh/download-center/home.html>
   - Realtek 网卡：<https://www.realtek.com/Download>
3. 将驱动文件复制到 U 盘，在目标电脑上安装。

#### 方法三：使用万能网卡驱动工具

可以提前下载 **驱动总裁（离线网卡版）** 或 **360 驱动大师网卡版** 等工具到 U 盘。这类工具内置了常见网卡驱动，可以离线安装网卡驱动来联网。

:::tip 提示
建议维修队成员在 U 盘中常备一份离线网卡驱动包，以应对各种机型。
:::

### 校园网连接

大部分高校校园网需要通过 Web 认证或专用客户端登录：

1. 有线连接：插入网线后，打开浏览器访问任意网页，通常会自动跳转到校园网认证页面。输入学号和密码即可登录。
2. 无线连接：搜索并连接校园 Wi-Fi（如 `NBUT-WiFi` 等），同样在浏览器中完成认证登录。
3. 如果需要专用客户端，按照学校网络中心的指引下载安装。

## 安装驱动和必要工具

联网之后，第一件事就是把所有硬件驱动装好，然后安装系统运行所需的基础组件。

### 安装硬件驱动

#### 方法一：通过品牌官网下载（推荐）

这是最安全、最可靠的方式。前往电脑品牌的官方支持页面，输入你的机型或序列号，下载对应的驱动包。

| 品牌 | 驱动下载页面                              |
| ---- | ----------------------------------------- |
| 联想 | <https://support.lenovo.com/>             |
| 华硕 | <https://www.asus.com.cn/support/>        |
| 惠普 | <https://support.hp.com/cn-zh/drivers>    |
| 戴尔 | <https://www.dell.com/support/home/zh-cn> |
| 宏碁 | <https://www.acer.com.cn/support.html>    |
| 华为 | <https://consumer.huawei.com/cn/support/> |
| 小米 | <https://www.mi.com/service/bijiben>      |

重点安装以下驱动（按优先级排序）：

1. **芯片组驱动**（Chipset）：主板芯片组的基础驱动，影响整体稳定性。
2. **显卡驱动**（Graphics）：
   - Intel 核显 / AMD 核显：到 Intel / AMD 官网下载。
   - NVIDIA 独显：<https://www.nvidia.cn/Download/index.aspx>
   - AMD 独显：<https://www.amd.com/zh-cn/support/download/drivers.html>
3. **声卡驱动**（Audio）：通常是 Realtek，从品牌官网下载最稳妥。
4. **触控板驱动**（Touchpad）：笔记本用户需要安装，否则手势功能无法使用。
5. **蓝牙驱动**（Bluetooth）：连接蓝牙耳机、鼠标等设备需要。
6. **其他**：指纹识别、摄像头、读卡器等，按需安装。

#### 方法二：通过 Windows Update 自动安装

Windows Update 会自动检测并安装大部分常见硬件驱动：

1. 打开 `设置` → `Windows 更新`。
2. 点击 **"检查更新"**，等待系统自动下载安装驱动。
3. 部分驱动可能在 **"可选更新"** 中，点击查看并手动选择安装。

:::warning 注意
Windows Update 提供的显卡驱动通常不是最新版本，游戏玩家建议从 NVIDIA / AMD 官网下载最新版驱动。
:::

#### 方法三：使用驱动管理工具

如果不确定自己的硬件型号，可以使用以下工具自动检测并安装：

- **Snappy Driver Installer Origin（SDIO）**：开源免费，无广告，支持离线驱动包。下载地址：<https://www.snappy-driver-installer.org/>
- **驱动总裁**：国产工具，内置大量驱动，适合批量装机。

:::warning 警告
**不要使用**"驱动人生""驱动精灵"等国产驱动工具的在线版本，它们通常会捆绑安装大量垃圾软件。如果非要使用，请选择离线版 / 网卡版，安装完驱动后立即卸载工具本身。
:::

### 安装运行库和基础组件

很多软件和游戏依赖以下运行库，建议全部安装：

#### Visual C++ Redistributable（VC++ 运行库）

大量软件和游戏依赖微软的 VC++ 运行库。推荐使用 **Visual C++ Redistributable AIO**（All in One）一键安装所有版本：

- GitHub 下载：<https://github.com/abbodi1406/vcredist>

或者手动从微软官网下载最新版本：<https://learn.microsoft.com/zh-cn/cpp/windows/latest-supported-vc-redist>

#### .NET Framework / .NET Runtime

部分软件需要 .NET 运行时：

- .NET Framework 3.5：在 `设置` → `应用` → `可选功能` → `更多 Windows 功能` 中勾选启用。
- .NET 8 / 9 Runtime：<https://dotnet.microsoft.com/zh-cn/download>

#### DirectX

游戏必备。大部分情况下 Windows 自带的 DirectX 已经足够，但部分老游戏可能需要额外安装 DirectX End-User Runtime：

- 下载地址：<https://www.microsoft.com/zh-cn/download/details.aspx?id=35>

#### 7-Zip 或其他解压工具

Windows 自带的解压功能较为有限，建议安装 **7-Zip**：

- 下载地址：<https://7-zip.org/>

7-Zip 免费开源，支持 `.7z`、`.rar`、`.tar`、`.gz` 等几乎所有压缩格式。

1. **从官网下载**：不要在百度搜索软件名后点击第一个"广告"结果，那通常是捆绑了垃圾软件的下载站。请直接访问软件官网。
2. **安装时注意选项**：安装过程中仔细阅读每一步，取消勾选"安装 XXX 浏览器""设 XXX 为主页"等捆绑选项。
3. **修改安装路径**：建议将软件安装到非系统盘（如 `D:\Software\`），减轻 C 盘压力。
4. **不要安装"全家桶"**：避免安装 360、腾讯电脑管家、2345 等国产"安全软件"，Windows Defender 已经足够好用。
   :::

## 个性化配置

系统装好、驱动和软件安装完毕后，可以进行一些个性化配置来提升日常使用体验。

### 任务栏和开始菜单

#### Windows 11

- **任务栏对齐方式**：`设置` → `个性化` → `任务栏` → `任务栏行为` → 任务栏对齐方式改为 **"靠左"**（如果你更习惯传统布局）。
- **隐藏不需要的图标**：在 `任务栏项` 中关闭"小组件""任务视图""聊天"等不需要的按钮。
- **显示秒数**：在 `任务栏行为` 中勾选"在系统托盘时钟中显示秒"。

#### Windows 10

- **隐藏搜索框**：右键任务栏 → `搜索` → 选择"隐藏"或"显示搜索图标"，节省任务栏空间。
- **隐藏 Cortana**：右键任务栏 → 取消勾选"显示 Cortana 按钮"。
- **小任务栏按钮**：`设置` → `个性化` → `任务栏` → 开启"使用小任务栏按钮"，可以显示更多图标。

### 关闭隐私数据收集

Windows 默认会收集大量使用数据，建议关闭：

1. 打开 `设置` → `隐私与安全性`（Win11）或 `设置` → `隐私`（Win10）。
2. **常规**：关闭"允许应用使用广告 ID""允许网站通过访问我的语言列表来提供本地相关内容"等所有选项。
3. **诊断和反馈**：将诊断数据设为"必需"（最低级别），关闭"改进墨迹书写和键入""量身定制的体验"。
4. **活动历史记录**：取消勾选"在此设备上存储我的活动历史记录"。

如果你使用的是专业版或企业版，还可以通过组策略进一步控制：

```powershell
# 打开组策略编辑器
gpedit.msc
```

在 `计算机配置` → `管理模板` → `Windows 组件` → `数据收集和预览版本` 中，可以将诊断数据设为最低级别。

### 电源计划优化

合理的电源设置可以在性能和续航之间取得平衡。

**笔记本用户：**

1. 打开 `设置` → `系统` → `电源和电池`（Win11）或 `电源和睡眠`（Win10）。
2. **插电时**：电源模式设为"最佳性能"；屏幕关闭时间根据个人喜好设置（建议 10~15 分钟）。
3. **使用电池时**：电源模式设为"平衡"或"最佳能效"；屏幕关闭时间设为 5 分钟。

**台式机用户：**

1. 打开 `控制面板` → `硬件和声音` → `电源选项`。
2. 选择 **"高性能"** 电源计划。
3. 如果看不到"高性能"选项，点击"创建电源计划"或在 PowerShell 中执行：

```powershell
# 启用高性能电源计划
powercfg /setactive 8c5e7fda-e8bf-4a96-9a85-a6e23a8c635c
```

:::tip 游戏玩家提示
如果你是游戏玩家，可以启用"卓越性能"电源计划（仅限专业版 / 企业版）：

```powershell
powercfg /duplicatescheme e9a42b02-d5df-448d-aa00-03f14749eb61
```

启用后在电源选项中选择"卓越性能"即可。
:::

### Windows Update 管理

Windows Update 默认会在后台自动下载和安装更新，有时会在不合适的时候重启电脑。以下是一些管理技巧：

1. **设置活动时间**：`设置` → `Windows 更新` → `高级选项` → `活动时间`，设置你常用电脑的时间段，Windows 不会在此期间自动重启。
2. **暂停更新**：如果近期有重要工作或考试，可以暂停更新最多 5 周。
3. **仅安装安全更新**：在 `高级选项` 中关闭"接收其他 Microsoft 产品的更新"和"获取最新更新"。

:::warning 不建议完全禁用更新
虽然网上有很多"彻底关闭 Windows Update"的教程，但 **强烈不建议这样做**。安全更新可以修复系统漏洞，保护你免受恶意软件攻击。建议只是调整更新策略（如暂停、设置活动时间），而不是完全禁用。
:::

### 文件资源管理器优化

1. 打开文件资源管理器，点击 `查看`（Win10 在功能区，Win11 点击顶部 `...`）→ `选项` → `文件夹选项`。
2. **常规**标签页：将"打开文件资源管理器时打开"改为 **"此电脑"**（默认是"快速访问"，对大多数人更实用的是直接看到各磁盘）。
3. **查看**标签页：
   - 取消勾选 **"隐藏已知文件类型的扩展名"**（这样你可以看到 `.exe`、`.docx` 等扩展名，有助于识别文件类型和防范伪装病毒）。
   - 选择 **"显示隐藏的文件、文件夹和驱动器"**。

### 默认应用设置

1. 打开 `设置` → `应用` → `默认应用`。
2. 将 **浏览器** 设为你安装的浏览器（Chrome / Firefox 等）。
3. 将 **视频播放器** 设为 PotPlayer 或其他你安装的播放器。
4. 将 **PDF 阅读器** 设为你偏好的工具（Edge 自带 PDF 阅读功能也不错）。

### 输入法配置

Windows 自带微软拼音输入法，基本够用。如果你习惯使用第三方输入法：

- **小狼毫（Rime）**：开源输入法框架，高度可定制，无广告无隐私问题。下载地址：<https://rime.im/>
- 如果使用搜狗、百度等输入法，建议在设置中关闭"广告推送""热点推荐"等功能，并关闭云联想等可能上传输入内容的功能。

:::tip 微软拼音小技巧
微软拼音输入法按 `Ctrl + Shift + B` 可以快速打开 Emoji 面板；按 `Win + .`（句号）也可以打开表情符号面板。按 `Ctrl + Shift + F` 可以切换简繁体输入。
:::
